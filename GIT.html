GIT Kommando Huskeliste

Grunnleggende GIT-kommandoer

git init
Oppretter et nytt git-repositorium

Detaljert Forklaring

  • git init: Initialiserer et nytt, tomt git-repo i gjeldende mappe.

Eksempel

git init

Brukes første gang du vil begynne å versjonere et prosjekt.

git clone <url>
Kopierer et eksternt repo til din maskin

Detaljert Forklaring

  • git clone: Laster ned hele repoet fra en ekstern kilde (f.eks. GitHub).

Eksempel

git clone https://github.com/bruker/prosjekt.git

Du får en lokal kopi av repoet med all historikk.

git status
Viser status for endringer i repoet

Detaljert Forklaring

  • git status: Viser hvilke filer som er endret, lagt til eller klare for commit.

Eksempel

git status

Gir deg oversikt over arbeidsområdet ditt.

git add <fil>
Legger til fil(er) i staging-området

Detaljert Forklaring

  • git add: Forbereder endringer for commit.

Eksempel

git add index.html

Endringene i index.html blir klare for commit.

git commit -m "Melding"
Lagrer endringer med en beskrivende melding

Detaljert Forklaring

  • git commit: Lagrer snapshot av endringene i repoet.

Eksempel

git commit -m "La til ny funksjon"

Bruk alltid en kort og beskrivende commit-melding.

git log
Viser commit-historikken

Detaljert Forklaring

  • git log: Viser en liste over tidligere commits.

Eksempel

git log

Gir deg oversikt over hvem som har gjort hva og når.

Arbeide med grener (branches)

git branch
Lister alle grener

Detaljert Forklaring

  • git branch: Viser alle grener i repoet og markerer den aktive.

Eksempel

git branch

Brukes for å se hvilke grener som finnes.

git checkout <gren>
Bytter til en annen gren

Detaljert Forklaring

  • git checkout: Bytter arbeidsområdet til valgt gren.

Eksempel

git checkout feature-login

Du kan nå jobbe på en annen funksjon uten å påvirke main.

git merge <gren>
Slår sammen en gren inn i aktiv gren

Detaljert Forklaring

  • git merge: Kombinerer endringer fra en annen gren inn i den du står på.

Eksempel

git merge feature-login

Endringene fra feature-login blir lagt til i aktiv gren.

git branch -d <gren>
Sletter en gren

Detaljert Forklaring

  • git branch -d: Sletter en lokal gren etter at den er ferdig.

Eksempel

git branch -d feature-login

Rydd opp etter at en funksjon er ferdig og merget.

Arbeide med eksternt repo

git remote -v
Viser eksterne repo-URLer

Detaljert Forklaring

  • git remote -v: Viser hvilke eksterne repoer som er koblet til.

Eksempel

git remote -v

Gir deg oversikt over hvor du kan pushe og hente fra.

git push
Laster opp endringer til eksternt repo

Detaljert Forklaring

  • git push: Sender dine commits til f.eks. GitHub.

Eksempel

git push origin main

Endringene dine blir tilgjengelige for andre.

git pull
Henter og slår sammen endringer fra eksternt repo

Detaljert Forklaring

  • git pull: Henter nye commits og oppdaterer din lokale gren.

Eksempel

git pull origin main

Hold ditt repo oppdatert med andres endringer.

git fetch
Henter endringer fra eksternt repo uten å slå dem sammen

Detaljert Forklaring

  • git fetch: Laster ned nye commits, men endrer ikke arbeidsområdet ditt.

Eksempel

git fetch origin

Brukes for å se hva som har skjedd eksternt før du eventuelt merger.

Andre nyttige kommandoer

git diff
Viser forskjeller mellom filer og commits

Detaljert Forklaring

  • git diff: Viser endringer som ikke er staged eller committed.

Eksempel

git diff

Se hva som er endret før du committer.

git stash
Lagrer midlertidig endringer for senere bruk

Detaljert Forklaring

  • git stash: Lagrer endringer som ikke er klare for commit, slik at du kan bytte gren.

Eksempel

git stash

Hent dem tilbake med git stash pop.

git reset --hard
Tilbakestiller repoet til siste commit

Detaljert Forklaring

  • git reset --hard: Forkaster alle lokale endringer.

Eksempel

git reset --hard

ADVARSEL: Endringer som ikke er commitet forsvinner!
