Enkle og Grunnleggende SQL-kommandoer

CREATE DATABASE database_navn;
Oppretter en ny database

Detaljert Forklaring

  • CREATE DATABASE: Brukes for å opprette en ny database.

Eksempel

CREATE DATABASE minDatabase;

Oppretter databasen "minDatabase" for lagring av tabeller og data.

SHOW DATABASES;
Viser alle tilgjengelige databaser

Detaljert Forklaring

  • SHOW DATABASES: Lister alle databaser som finnes på serveren.

Eksempel

SHOW DATABASES;

Brukes for å se hvilke databaser som er tilgjengelige, spesielt i MySQL.

USE database_navn;
Velger hvilken database som skal brukes

Detaljert Forklaring

  • USE: Angir hvilken database som skal være aktiv for påfølgende spørringer.

Eksempel

USE minDatabase;

Setter "minDatabase" som aktiv database for videre operasjoner.

DESCRIBE tabell;
Viser strukturen til en tabell

Detaljert Forklaring

  • DESCRIBE: Gir en oversikt over kolonner, datatyper, null-verdier og nøkler for en tabell.

Eksempel

DESCRIBE ansatte;

Brukes for å se strukturen til tabellen "ansatte".

TRUNCATE TABLE tabell;
Fjerner alle rader i en tabell hurtig

Detaljert Forklaring

  • TRUNCATE TABLE: Raskt fjerner alle rader i en tabell, ofte raskere enn DELETE siden den ikke logger hver rad.

Eksempel

TRUNCATE TABLE ansatte;

Fjerner alle data fra tabellen "ansatte" og tilbakestiller eventuelle auto-inkrementverdier.

RENAME TABLE gammelt_navn TO nytt_navn;
Endrer navnet på en eksisterende tabell

Detaljert Forklaring

  • RENAME TABLE: Endrer navnet på tabellen.

Eksempel

RENAME TABLE gammelAnsatte TO ansatte;

Endrer tabellnavnet fra "gammelAnsatte" til "ansatte".

EXPLAIN SELECT ...;
Gir innsikt i hvordan en spørring kjøres

Detaljert Forklaring

  • EXPLAIN: Viser spørringens utførelsesplan, nyttig for optimalisering og debugging.

Eksempel

EXPLAIN SELECT * FROM ansatte;

Analyserer hvordan MySQL henter data for spørringen "SELECT * FROM ansatte".

SELECT * FROM tabell;
Henter alle kolonner fra en tabell

Detaljert Forklaring

  • SELECT: Nøkkelord som angir at vi vil hente data
  • *: Betyr "alle kolonner"
  • FROM: Spesifiserer hvilken tabell vi henter fra

Eksempel

SELECT * FROM ansatte;
-- Henter alle kolonner og rader fra ansatte-tabellen

Bruk denne når du vil se all informasjon i en tabell. Vær forsiktig med store tabeller!

SELECT kolonne FROM tabell WHERE betingelse;
Henter spesifikke data med filter

Detaljert Forklaring

  • SELECT kolonne: Velg spesifikke kolonner i stedet for alle (*)
  • WHERE: Setter betingelser for hvilke rader som skal hentes
  • betingelse: For eksempel alder > 25, navn = 'Ole', etc.

Eksempel

SELECT fornavn, etternavn 
FROM ansatte 
WHERE alder > 30;

Brukes når du vil filtrere data og bare se spesifikke kolonner.

Data Manipulering

INSERT INTO tabell (kolonne1, kolonne2) VALUES (verdi1, verdi2);
Legger til ny rad i tabellen

Detaljert Forklaring

  • INSERT INTO: Angir at vi vil legge til data
  • (kolonne1, kolonne2): Liste over kolonner vi vil fylle
  • VALUES: Verdiene som skal settes inn

Eksempel

INSERT INTO ansatte (fornavn, etternavn, stilling) 
VALUES ('Ole', 'Hansen', 'Utvikler');

Husk å inkludere alle påkrevde felter når du legger til nye rader.

UPDATE tabell SET kolonne = verdi WHERE betingelse;
Oppdaterer eksisterende data

Detaljert Forklaring

  • UPDATE: Angir at vi vil endre eksisterende data
  • SET: Spesifiserer hvilke verdier som skal endres
  • WHERE: Viktig! Definerer hvilke rader som skal oppdateres

Eksempel

UPDATE ansatte 
SET stilling = 'Senior Utvikler' 
WHERE id = 1;

VIKTIG: Alltid bruk WHERE-betingelse med UPDATE for å unngå å endre alle rader!

DELETE FROM tabell WHERE betingelse;
Sletter rader fra tabellen

Detaljert Forklaring

  • DELETE FROM: Angir at vi vil slette data
  • WHERE: Kritisk! Definerer hvilke rader som skal slettes

Eksempel

DELETE FROM ansatte 
WHERE slutt_dato < '2023-01-01';

ADVARSEL: Alltid bruk WHERE-betingelse med DELETE for å unngå å slette alle rader!

Tabelloperasjoner

CREATE TABLE tabell (kolonne1 type, kolonne2 type);
Oppretter ny tabell

Detaljert Forklaring

  • CREATE TABLE: Lager en ny tabell
  • type: F.eks. INTEGER, VARCHAR(255), DATE, etc.

Eksempel

CREATE TABLE ansatte (
    id INTEGER PRIMARY KEY,
    fornavn VARCHAR(50),
    etternavn VARCHAR(50),
    ansatt_dato DATE
);

Planlegg tabellstrukturen nøye før du oppretter den.

ALTER TABLE tabell ADD kolonne type;
Legger til ny kolonne i eksisterende tabell

Detaljert Forklaring

  • ALTER TABLE: Modifiserer en eksisterende tabell
  • ADD: Legger til en ny kolonne

Eksempel

ALTER TABLE ansatte 
ADD epost VARCHAR(100);

Nye kolonner legges til på slutten av tabellen. Vær obs på at eksisterende rader får NULL-verdi i den nye kolonnen.

DROP TABLE tabell;
Sletter en tabell

Detaljert Forklaring

  • DROP TABLE: Sletter tabellen og all data i den

Eksempel

DROP TABLE gamle_ansatte;

ADVARSEL: Dette kan ikke angres! Sørg for å ha backup før du sletter tabeller.

Joins

SELECT * FROM tabell1 INNER JOIN tabell2 ON tabell1.id = tabell2.id;
Kobler sammen to tabeller basert på felles verdier

Detaljert Forklaring

  • INNER JOIN: Matcher rader fra begge tabeller
  • ON: Definerer koblingsbetingelsen

Eksempel

SELECT ansatte.navn, avdelinger.avdeling_navn
FROM ansatte
INNER JOIN avdelinger ON ansatte.avdeling_id = avdelinger.id;

Viser kun rader som har match i begge tabeller.

SELECT * FROM tabell1 LEFT JOIN tabell2 ON tabell1.id = tabell2.id;
Beholder alle rader fra venstre tabell

Detaljert Forklaring

  • LEFT JOIN: Beholder alle rader fra første tabell
  • ON: Definerer koblingsbetingelsen

Eksempel

SELECT ansatte.navn, prosjekter.prosjekt_navn
FROM ansatte
LEFT JOIN prosjekter ON ansatte.prosjekt_id = prosjekter.id;

Viser alle ansatte, selv om de ikke er tilknyttet et prosjekt.

Avanserte Spørringer og Funksjoner

SELECT DISTINCT kolonne FROM tabell;
Henter unike verdier fra en kolonne

Detaljert Forklaring

  • DISTINCT: Fjerner dupliserte verdier i resultatet

Eksempel

SELECT DISTINCT land FROM kunder;

Brukes for å hente en liste over unike land fra kunder-tabellen.

SELECT * FROM tabell ORDER BY kolonne ASC|DESC;
Sorter resultatet basert på en kolonne

Detaljert Forklaring

  • ORDER BY: Sorterer data basert på spesifisert kolonne
  • ASC: Stigende rekkefølge (default)
  • DESC: Synkende rekkefølge

Eksempel

SELECT * FROM produkter ORDER BY pris DESC;

Brukes for å vise de dyreste produktene først.

SELECT kolonne, COUNT(*) FROM tabell GROUP BY kolonne;
Grupperer data etter en kolonne og teller antallet

Detaljert Forklaring

  • GROUP BY: Samler rader med samme verdi i en kolonne
  • COUNT(*): Teller antall rader i hver gruppe

Eksempel

SELECT kategori, COUNT(*)
FROM produkter
GROUP BY kategori;

Brukes for å vise antall produkter per kategori.

SELECT kolonne, COUNT(*) FROM tabell GROUP BY kolonne HAVING COUNT(*) > verdi;
Filtrerer grupper med en betingelse

Detaljert Forklaring

  • HAVING: Filtrerer grupper etter at de er gruppert med GROUP BY

Eksempel

SELECT kategori, COUNT(*)
FROM produkter
GROUP BY kategori
HAVING COUNT(*) > 10;

Brukes for å vise kun de kategoriene som har mer enn 10 produkter.

SELECT * FROM tabell LIMIT antall;
Begrenser antall rader i resultatet

Detaljert Forklaring

  • LIMIT: Bestemmer hvor mange rader som returneres

Eksempel

SELECT * FROM bestillinger LIMIT 5;

Brukes ofte for paginering eller når du bare trenger et utvalg av data.

SELECT kolonne FROM tabell1 UNION SELECT kolonne FROM tabell2;
Kombinerer resultater fra to spørringer

Detaljert Forklaring

  • UNION: Kombinerer resultater og fjerner duplikater
  • UNION ALL: Kombinerer resultater uten å fjerne duplikater

Eksempel

SELECT fornavn FROM ansatte
UNION
SELECT fornavn FROM kunder;

Brukes for å lage en liste med unike fornavn fra både ansatte og kunder.

CREATE INDEX index_navn ON tabell(kolonne);
Oppretter en indeks for å øke søkehastigheten i en tabell

Detaljert Forklaring

  • CREATE INDEX: Lager en indeks på en eller flere kolonner

Eksempel

CREATE INDEX idx_pris ON produkter(pris);

Brukes for å forbedre ytelsen ved søk i den spesifiserte kolonnen.

DROP INDEX index_navn ON tabell;
Fjerner en indeks fra en tabell

Detaljert Forklaring

  • DROP INDEX: Fjerner en indeks

Eksempel

DROP INDEX idx_pris ON produkter;

Fjerner indeksen for å frigjøre ressurser eller endre tabellstrukturen.

Flere SQL-kommandoer

SELECT * FROM ansatte WHERE lønn = (SELECT MAX(lønn) FROM ansatte);
Henter den ansatte med høyeste lønn ved hjelp av en subquery

Detaljert Forklaring

  • Subquery: En spørring som returnerer et resultat som benyttes av hovedspørringen

Eksempel

SELECT * FROM ansatte 
WHERE lønn = (SELECT MAX(lønn) FROM ansatte);
-- Henter den ansatte med høyeste lønn

Subqueryen finner den høyeste lønnen, og hovedspørringen returnerer den ansatte som har denne lønnen.

WITH avdelingslønn AS ( SELECT avdeling, AVG(lønn) AS gjennomsnittslønn FROM ansatte GROUP BY avdeling ) SELECT * FROM avdelingslønn;
Bruker en Common Table Expression (CTE) for å beregne gjennomsnittslønn per avdeling

Detaljert Forklaring

  • WITH: Definerer en midlertidig resultattabell (CTE) som kan benyttes i den påfølgende spørringen

Eksempel

WITH avdelingslønn AS (
    SELECT avdeling, AVG(lønn) AS gjennomsnittslønn
    FROM ansatte
    GROUP BY avdeling
)
SELECT * FROM avdelingslønn;
-- Viser gjennomsnittslønn for hver avdeling

CTE gjør komplekse spørringer lettere å lese og vedlikeholde.

CREATE VIEW aktive_kunder AS SELECT * FROM kunder WHERE aktiv = 1;
Oppretter en view for aktive kunder

Detaljert Forklaring

  • CREATE VIEW: Lager en virtuell tabell basert på resultatet fra en SELECT-spørring

Eksempel

CREATE VIEW aktive_kunder AS 
SELECT * FROM kunder 
WHERE aktiv = 1;
-- Lager en view som kun inneholder aktive kunder

Views forenkler komplekse spørringer og kan også begrense tilgang til sensitive data.

DROP VIEW aktive_kunder;
Fjerner viewen for aktive kunder

Detaljert Forklaring

  • DROP VIEW: Sletter en eksisterende view

Eksempel

DROP VIEW aktive_kunder;
-- Fjerner viewen for aktive kunder

Brukes for å fjerne views som ikke lenger er nødvendige.

SELECT navn, lønn, CASE WHEN lønn < 50000 THEN 'Lav' WHEN lønn BETWEEN 50000 AND 100000 THEN 'Middels' ELSE 'Høy' END AS lønnsklasse FROM ansatte;
Bruker CASE for å kategorisere ansatte basert på lønn

Detaljert Forklaring

  • CASE: Evaluerer en rekke betingelser og returnerer den første sanne verdien

Eksempel

SELECT navn, lønn,
CASE 
    WHEN lønn < 50000 THEN 'Lav'
    WHEN lønn BETWEEN 50000 AND 100000 THEN 'Middels'
    ELSE 'Høy'
END AS lønnsklasse
FROM ansatte;
-- Kategoriserer ansatte basert på deres lønn

CASE-setningen gir mulighet for dynamisk evaluering og visning av data basert på spesifikke betingelser.

SELECT navn, lønn, ROW_NUMBER() OVER (ORDER BY lønn DESC) AS rangering FROM ansatte;
Bruker window-funksjonen ROW_NUMBER for å rangere ansatte etter lønn

Detaljert Forklaring

  • ROW_NUMBER() OVER (ORDER BY): Genererer et unikt radnummer for hver rad basert på den oppgitte rekkefølgen

Eksempel

SELECT navn, lønn,
ROW_NUMBER() OVER (ORDER BY lønn DESC) AS rangering
FROM ansatte;
-- Rangering av ansatte med den høyeste lønnen først

Window-funksjoner gjør det mulig å utføre avanserte beregninger over et sett med rader uten å aggregere dem.
